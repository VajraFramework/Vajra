1. Windows

Note that most of the directions below apply to both 32bit and 64bit windows except where specifically noted otherwise.
!! However !!, the precompiled binaries and libraries which are mentioned below are applicable only for Win64 (if you are using Win32, you'll have to compile them from source yourself)

a. Installing required tools:

Step 1: The MinGW compiler suite:
We'll use the MinGW (Minimalist GNU for Windows) suite of compilers and other tools.
Note that while you can get it from the official page at http://www.mingw.org, for convenience there is a preinstalled version provided here: MinGW.tar.gz
It is highly recommended that you download this copy (since it comes pre-installed with all the libraries needed for building Vajra on Windows64)

Extract the downloaded MinGW folder into any location on your hard drive (preferably C:\ since that is a standard path, although it isn't required that it be in C:\ for Vajra)
Assuming you extracted it to C:\ (which you really should!) your folder structure should have C:\MinGW\bin\

Now, add "C:\MinGW\bin\" to your system's Path variable. To verify, at this point you should be able to go into the windows cmd prompt and type "mingw32-g++.exe -v", and it shouldn't tell you that no such program exists. A restart *may* be required after setting the path, although it shouldn't be.

Alright. We have MinGW.

Step 2: Eclipse for C++:
We'll be using Eclipse with the CDT plugin for C++ development. You can download Eclipse pre-bundled with CDT here: Eclipse Kepler with CDT

Step 3: Cygwin:
Cygwin provides a linux-like command-line environment in windows. Its awesome. You can get the cygwin setup file here Cygwin 32 bit Installer
!! Note !! The link above reads "32bit cygwin installer". Its not a typo.
Do NOT install the x86_64 installer which is also available via Cygwin's website.

Run the installer, and preferably stick to all the default settings until you arrive at the page called "Cygwin Setup - Select Packages"
Leave everything at default (because the defaults are good enough), but specifically search for a package called "rsync" (in the little search text box in the top-left) and mark that package to be installed.
Finish the rest of the installation (it might take a long time for cygwin to finish installing)
One more thing: Assuming you installed cygwin to the default location of C:\cygwin, add "C:\cygwin\usr\bin\" to your system's Path variable.

To verify, at this point you should be able to go into the windows cmd prompt and type "bash.exe", and it shouldn't tell you that no such program exists. A restart *may* be required after setting the path, although it shouldn't be.

Alright. We have Cygwin.

b. Downloading the source code:

Step 1: Read about git:
Real talk. Spend at least 30minutes to review git concepts. You won't regret it.

Step 2: Download git tools of your choice:
Now, there are a lot of git GUI tools available out there and you may use any of those. However, it is highly useful to also have a git command line to fall back on (at least for the first step (below)). I highly recommend getting this: Git for windows
When you are done installing that (with all the default options), you should have "Git Bash" installed on your computer. Fire up Git Bash; you should be at a minimalistic command prompt. Navigate to a folder of your choice and clone the repository:

> git clone https://github.com/VajraFramework/Vajra.git

This, should clone the main repository in your current location. However, before you can build, you need to get some of the Tools/ needed to build.
These tools are stored as a git "submodule" and in order to get them you need to step into the newly cloned Vajra/ folder and init an dupdate the submodule, like so:

> cd Vajra/
> git submodule init
> git submodule update

Feel free to use any GUI frontend for git to make things easier from this point onwards.


c. Building the source code (Finally!):

Step 1: From the windows cmd prompt:
Just this once, we'll build it from the cmd line. Feel free to skip this step and move on to Step 2 below which uses eclipse.
Open up the windows cmd prompt and navigate to the folder where you cloned the repository and type:

> build.bat windows

Note that just typing "build.bat" will print usage information and such.

Step 2: Using Eclipse:
Launch the Eclipse you downloaded. When prompted, set the workspace to be the top level folder ("Vajra") of the repository on your hard disk.
This will init an empty workspace. Now, import (File->Import) "Existing Project into workspace" and point it to the "VajraFramework" folder inside Vajra/

This should load up the project inside the workspace with pretty much everything configured. Note that there will be multiple named build configurations (Linux, Windows, and Android)
At this point, you can select the "Windows" build configuration and it should get built.

Similarly, there should be multiple Run/Debug configurations. Select the "VajraFramwwork Run" (or Debug) configuration to run (or debug) the application.
You should be able to use all your standard debugging tools from the toolset, or use the built-in gdb console.
